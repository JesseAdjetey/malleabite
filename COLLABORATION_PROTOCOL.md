# Collaboration Protocol

**Last Updated:** March 2, 2026  
**Purpose:** Define the working approach for logic-related tasks and decisions

---

## Core Working Approach

For **every logic-related request**, I will follow this workflow:

### 1. **Analyze & Understand** ✓
- Deeply understand what you're asking for
- Ask clarifying questions if needed
- Get full context before proposing solutions

### 2. **Brainstorm Multiple Solutions** ✓
- Come up with multiple in-depth approaches
- Think through different strategies
- Consider various implementation paths

### 3. **Critical Analysis** ✓
- Evaluate pros and cons of each solution
- Identify potential problems with each approach
- Consider edge cases and maintainability
- Think through technical implications

### 4. **Present Options** ✓
- **Always present multiple options**, not just my top choice
- Clearly mark which option I think is best and why
- Explain trade-offs for each approach
- Be transparent about uncertainties

### 5. **Get Your Input** ✓
- Ask you to choose from the presented options
- Respect your decision even if different from my recommendation
- Only proceed with implementation after you confirm the direction

---

## What This Means Practically

- **No assumptions**: I won't assume you want my single recommended path
- **Visibility**: You'll always see my reasoning and alternatives
- **Agency**: You decide which direction to go
- **Quality**: Better decisions through collaborative thinking
- **Speed**: Clarifying upfront saves time on rework

---

## Scope

This protocol applies to:
- Feature architecture decisions
- Logic implementation approaches
- System design questions
- Problem-solving tasks
- Trade-off decisions

---

## Quick Reference

If I ever:
- Jump straight to implementation without presenting options → Call me out
- Don't show my reasoning → Point to this document
- Don't ask for your input → Remind me to follow the protocol

You can simply reference this document or say: *"Let's follow the protocol"*

## Animation Philosophy

**Your Animation Style:**
- Intentional, meaningful, subtle, simple
- Tactile, realistic physics-based (not artificial)
- Gradients that are subtle (5-10% opacity changes max)
- Elements feel like they have physical mass and weight
- **Refraction effects:** Like dragging something from air into water
  - Example: Moving module element to calendar = color/opacity shifts, blur changes, shadow depth increases
  - Smooth medium transitions (200-300ms)
  - Never jarring or sudden

**When I propose animations:**
- I'll consider physics-realistic motion
- Spring damping over linear easing
- Subtle over flashy
- Every animation must serve a purpose
- Recommend 200-400ms durations
- Include gradient shifts where appropriate
- Think "tactile feedback" not "cool effects"
